# Chaos Engineering with Playwright

## Overview

This project demonstrates how **Playwright** can be integrated with **Chaos Mesh** to automate Chaos Engineering scenarios from the QA automation framework.

The framework allows QA engineers to:

- Execute normal UI automation
- Trigger Chaos Mesh experiments
- Validate application resilience
- Validate business workflows during infrastructure failures
- Automatically clean up chaos experiments

# Install Chaos Mesh

Add repository

```bash
helm repo add chaos-mesh https://charts.chaos-mesh.org

helm repo update
```

Create namespace

```bash
kubectl create namespace chaos-mesh
```

Install

```bash
helm install chaos-mesh chaos-mesh/chaos-mesh \
--namespace chaos-mesh \
--set chaosDaemon.runtime=docker \
--set chaosDaemon.socketPath=/var/run/docker.sock
```

Verify

```bash
kubectl get pods -n chaos-mesh
```

# Notes

- Keep the terminal running `minikube service frontend-external --url` open while executing Playwright tests.
- Update the `BASE_URL` in `.env` whenever Minikube assigns a new URL.
- Ensure all Kubernetes pods are in the `Running` state before executing Chaos tests.
- Chaos Mesh must be installed before running any Chaos Engineering test cases.
- If a Chaos experiment already exists or has been removed, the framework handles cleanup automatically.
